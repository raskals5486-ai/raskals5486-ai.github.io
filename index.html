Для кого

Для водителей такси в Санкт-Петербурге, которым нужен компактный информационный ориентир при планировании рабочего времени и перемещений по городу.

Чего сервис не делает

Не принимает заказы, не гарантирует доход или повышающий коэффициент, не управляет приложениями агрегаторов и не получает данные из Яндекс Про.

Для проекта запрашиваются clid и API key к API информации о поездке. Планируется серверное получение данных по заранее определённым маршрутам от транспортных узлов Санкт-Петербурга.

  • показ ориентировочной стоимости и времени поездки конечному пользователю;
  • тарифы «Эконом», «Комфорт» и «Комфорт+» — в пределах доступности API;
  • ключ хранится только на backend и не включается в APK;
  • частота запросов и хранение истории — только в согласованных Яндексом пределах;
  • никакого сравнения с конкурентами и автоматического заказа поездок.

03

Ответственная разработка

Данные, безопасность и прозрачность

01

Минимизация данных

Не запрашиваются ФИО, телефоны, маршруты конкретных пассажиров, сведения о заказах, оплате или водителях.

02

Защита ключа

Ключи внешних API размещаются в серверном хранилище секретов. Мобильное приложение получает только подготовленный ответ backend.

03

Разрешение источника

Любой live-источник включается только после проверки условий и получения необходимого разрешения или договора.

04

Честная маркировка

При отсутствии свежих данных интерфейс показывает «Нет данных», а не подменяет это низким уровнем спроса.
